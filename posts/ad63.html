1反射和注解

1.1反射

1.1.1反射的引入

  1. 在程序运行时动态加载类并获取类的详细信息，从而更好的操作类或对象的属性和方法
  2. 使用反射机制可以动态获取当前class的信息 比如方法的信息、注解信息、方法的参数、属性等
  3. 目的：提供开发者能够更好封装框架实现扩展功能

1.1.2反射优缺点

  1. 优点：
    • 可以实现动态创建对象和编译，体现出很大的灵活性
  2. 缺点
    • 运行时告诉JVM想要做什么总是慢于直接直接执行
    • 反射调用方法时可以忽略权限检查，因此可能会破坏封装性而导致安全问题点击跳转

        Employee employee1 = new Employee();
        Class<? extends Employee> aClass = employee1.getClass();
        Class<Employee> employeeClass = Employee.class;
        Class<?> aClass1 = Class.forName("Reflection_.Fields_AND_Method.Employee");
        System.out.println(aClass1==aClass);//true
        System.out.println(aClass==employeeClass);//true 编译Employee类后只会有一个.class文件，
        //JVM为每一个类创建只创建一个Class对象，不管是foName（），还是getClass（）或者.class获得到的对象变量都指向
        // 同一个Class对象
        //forName（）方法使用较多

        Employee instance = aClass.newInstance();
        //由于aClass由无参的getClass（）获得，调用newInstance（）将调用Employee类的无参构造器,若类中没有无参构造方法，将抛出异常
        //java9以后该方法已被废弃了

        System.out.println(instance==employee1);//false，由反射构建的类实例对象和类构造器构建的对象内存地址不同
        System.out.println(instance); //out:  Employee{name='null', age=0}



        Class<?> aClass2 = Class.forName("Reflection_.Fields_AND_Method.Employee");
        var constructor = aClass2.getDeclaredConstructor(String.class, int.class);
        Employee instance1 = (Employee)constructor.newInstance("姜", 14);//Employee{name='姜', age=14}
        System.out.println(instance1);

        //getField,getConstructor,getMethod()将会返回按当前类，实现接口，父类顺序
        // 且为public标识查找，若无，抛出NoSuchFieldException
        Field heer = aClass2.getField("sex");
        System.out.println(heer);//public boolean Reflection_.Fields_AND_Method.Employee.sex

        Constructor<?> constructor1 = aClass2.getConstructor(String.class, int.class);
        System.out.println(constructor1);//public Reflection_.Fields_AND_Method.Employee(java.lang.String,int)
        System.out.println("------------------------");

        // .getFields、getMethods和getCostructors方法
        // 可以获得类提供的public域、方法和构造器数组，其中包括超类的共有成员
        Field[] fields = aClass2.getFields();
        for (Field field : fields) {
            System.out.println(field);
        }
        //getDeclatedFields、getDeclatedMethods和getDeclaredConstructors方法
        // 可以获得类中声明的全部域、方法和构造器，其中包括私有和受保护的成员，但不包括超类的成员。

        //可以用反射为属性赋值
        Field name = aClass2.getDeclaredField("name");
        Employee o = (Employee) aClass2.newInstance();
        name.setAccessible(true);//由于在Employee类中的name属性为private则修改访问权限,否则IllegalAccessException
        name.set(o,"jack");
        System.out.println(o);//Employee{name='jack', age=0}

        //利用反射调用方法
        Method method = aClass2.getDeclaredMethod("reflectionMethod", int.class, String.class);
        method.setAccessible(true);//私有方法，需要重新设置权限
        Object invoke = method.invoke(o, 666, "反射方法");
        System.out.println(invoke);
        
    }
}

1.2注解

1.2.1注解的概念

1.注解是一种元数据形式，同类，接口，基本数据类型，都为java的一种数据类型
2.可用标注类，接口，变量，方法，包，方法，参数
3.同Javadoc的区别：

  • 注解支持自定义，可以工作于编译.class文件，以及运行时

1.2.2注解的依赖关系

####每个annatation都有一个RetentionPolicy属性（SOURCE，CLASS，RUNTIME），若干个ElementType属性（TYPE等）

public enum RetentionPolicy {
    /**
     * 能被编译器检查
     */
    SOURCE,

    /**
     *被编译器保存至字节码文件中，但不会保持至VM运行时，是默认的注解属性
     */
    CLASS,

    /**
      * 注释将由编译器记录在类文件中，并在运行时由虚拟机保留。 在运行时由虚拟机保留，因此它们可以被反射    *性地读取。
     * @see java.lang.reflect.AnnotatedElement
     */
    RUNTIME
}

public enum ElementType {
    /** Class, interface (including annotation type), or enum declaration .annotation type则可以嵌套使用*/
    TYPE,

    /** Field declaration (includes enum constants) */
    FIELD,

    /** Method declaration */
    METHOD,

    /** Formal parameter declaration */
    PARAMETER,

    /** Constructor declaration */
    CONSTRUCTOR,

    /** Local variable declaration */
    LOCAL_VARIABLE,

    /** Annotation type declaration */
    ANNOTATION_TYPE,

    /** Package declaration */
    PACKAGE,

    /**
     * Type parameter declaration
     *
     * @since 1.8
     */
    TYPE_PARAMETER,

    /**
     * Use of a type
     *
     * @since 1.8
     */
    TYPE_USE,

    /**
     * Module declaration.
     *
     * @since 9
     */
    MODULE
}

####自定义注解

需求：

有一个Employee类，该类有一个方法注解用于获取Employee对象的一些属性；有一Out类中的方法被Employee中的注解标注，该方法用于打印Employee对象信息。编写一个测试类，利用反射检测Employee类中是否存在方法注解，若存在，则打印当前的对象信息；反之，则输出不存在任何方法注解。

Employee类：

package Annotation_;

import java.lang.annotation.*;
import java.util.ArrayList;

/**
 * @author DevilStory
 * @version 0.0.1
 * @Date 2021/7/31
 */
@Target(ElementType.METHOD)//该注解用于方法
@Documented//可生成与Javadoc中
@Retention(RetentionPolicy.RUNTIME)//该注解作用至代码运行时，故可与反射同时使用
@interface infor{
    String name() default "null";
    int age() default 0;
    String[] habits() default {"null"};//为字符串数组

}

public class Employee {
    private String name;
    private int age;
    private ArrayList habits;//ArrayList属性



    public Employee(){}


    public Employee(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public ArrayList getHabits() {
        return habits;
    }

    public void setHabits(ArrayList habits) {
        this.habits = habits;
    }
}

测试类：

package Annotation_;

import java.lang.reflect.InvocationTargetException;
import java.lang.reflect.Method;
import java.util.ArrayList;
import java.util.Arrays;

/**
 * @author DevilStory
 * @version 0.0.1
 * @Date 2021/7/31
 */
public class AnnotationTest {
    public static void main(String[] args) throws ClassNotFoundException,
            NoSuchMethodException, InvocationTargetException, IllegalAccessException {
        //Employee的Class对象
        Class<Employee> s = Employee.class;
        //Out的Class对象
        Class<?> aClass = Out.class;
        //利用反射获得Out中的“out_for”方法的Method对象，
        //public Method getDeclaredMethod(String name, Class<?>... parameterTypes)
        //第二个形参带入s，否则会出现NoSuchMethodException，由于该注解定义在Employee类中
        Method outMethod = aClass.getDeclaredMethod("out_for", s);

        //outMethod对象（Out类中out_for方法）是否存在于@infor的注解
        if(!outMethod.isAnnotationPresent(infor.class)){
            System.out.println("no such @infor innotation");
            return;

        }
        Employee employee = new Employee();
        //获得infor注解的对象
        infor annotation = outMethod.getAnnotation(infor.class);
        //注解对象.注解属性的方式获得属性值 annotation.name()
        employee.setName(annotation.name());
        employee.setAge(annotation.age());
        //由于Employee类中的属性habits为ArrayList，而注解属性habits()为String[]，且不能为ArrayList[]
        //故调用Arrays.asList（）方法进行转换
        employee.setHabits(new ArrayList(Arrays.asList(annotation.habits())));
        Out out = new Out();
        //利用Method类中invoke（）执行Out类中out_for（）方法
        outMethod.invoke(out,employee);
    }


}

1.2.3.5注解理解

  1. 所有的自定义注解都实现了Annotation接口。即为接口 extends 接口

  2. 当创建注解类的实例时，如getAnnotation（）方法，JDK 是通过动态代理机制生成一个实现我们注解（接口）的代理类
