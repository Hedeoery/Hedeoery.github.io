#简单并发控制

0.1并发问题

当同一个对象被多个线程同时操作时，产生数据安全问题

##并发问题根源

0.1.1原子性

  • 操作系统增加了进程、线程，以分时复用 CPU，进而均衡 CPU 与 I/O 设备的速度差异；// 导致 原子性问题

0.1.2可见性

  • CPU 增加了缓存，以均衡与内存的速度差异；// 导致 可见性问题

0.1.3有序性

  • 编译程序优化指令执行次序，使得缓存能够得到更加合理地利用。// 导致 有序性问题

0.2解决方法

###互斥同步

  • 采用加锁（synchronized 和 ReentrantLock）
  1. 加锁机制：使对象拥有锁，独占资源，只有等待对象释放锁后，其他线程才可以享受该资源
  2. 加锁存在的问题：
    • 一个线程加锁会导致其他需要该锁的线程被挂起
    • 加锁和释放锁导致的上下文切换和调度延时，引起性能问题
    • 高优先级的线程被低优先级的线程的占有锁导致优先级倒置，引起性能问题。点击跳转线程优先级

0.2.1非阻塞同步(需要后期补充)

  • CAS
  • AtomicInteger
  • ABA

    0.2.2一些不需要同步的线程(需要后期补充)

  • 栈封闭
  • 线程本地存储(Thread Local Storage)
  • 可重入代码(Reentrant Code)

采用添加关键字synchronized

0.3.1同步方法

  • 对成员方法，即对this对象加锁，方法中用到的对象都加锁
  • 静态方法，对该方法中使用到的反射对象加锁

0.3.2.1存在问题

  1. 对一个大的方法添加synchronized，导致性能下降，大多数情况采用同步块
  2. 同步块使用不当，造成同步问题仍然存在、、
  3. 同步块锁住的为内存地址不变的对象？？
  4. 应考虑高并发，高性能，高实现，即安全性能均需考虑

package thread_.safeproblem;

/**采用两台机器抢票，票共10张
 * 考虑多线程同步问题，用同步方法和同步块两种方式解决问题
 * @author DevilStory
 * @version 0.0.1
 * @Date 2021/8/11
 */
public class SafeTest01 {
    public static void main(String[] args) {
        Ticket tickets = new Ticket();
        //tickets对象同时被2个线程操作买票，会产生数据的安全问题
        Thread thread01 = new Thread(tickets, "一号机");
        Thread thread02 = new Thread(tickets, "二号机");
        thread01.start();
        thread02.start();
    }
}

 * 售票处

class Ticket implements Runnable{
    private int sum=10;
    private boolean flag=true;
    
    @Override
    public void run() {
        while (flag) {
            test();
        }
    }
    
//同步成员方法（test），即this对象加锁，方法操作的对象加锁，本例中为sum和flag
    private /*synchronized*/ void test() {

        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
//        考虑性能问题，当票数为0时，不需要进入同步块
        if(sum==0){
            return;
        }

//        同步块
        synchronized (this) {
            if(sum<=0){
            flag=false;
            return;
            }

            System.out.println(Thread.currentThread().getName()+sum);
            sum--;
        }
    }
}

结果并非一定按照设置的优先级执行

原因：线程的优先级设置可以理解为线程抢占CPU时间片的概率，虽然概率比较大，但是它不一定就是按照优先级的顺序去抢占CPU时间片的，具体的执行顺序还是要根据谁先抢到了CPU的时间片，谁就先来执行。

###yield（）方法

package thread_.threadmethod_;


/**线程的常见方法演示
 * sleep，start，join，yield，wait，
 * @author DevilStory
 * @version 0.0.1
 * @Date 2021/8/12
 */

public class NormalMethod {
    public static void main(String[] args)throws InterruptedException {

          NormalMethod.yield_();

        for (int i = 0; i < 3; i++) {
            Thread.sleep(1000);
            System.out.println("Main");
        }
    }

    /**
     * yield()，a线程调用，则a礼让其他线程，a进入Ready状态，不释放锁，和其他线程等待CPU调度
     */
    public static void yield_() {

        new Thread(() -> {
            for (int i = 0; i < 3; i++) {
                //礼让main线程
                Thread.yield();
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("yield");
            }
        }).start();

    }
}

###sleep（）

public class NormalMethod {
    public static void main(String[] args)throws InterruptedException {
        NormalMethod.sleep_();
    }
    
/**
     * sleep(sleeptime),线程a调用此方法，a进入TIMED WAITING，不释放锁，当其他线程需要a线程的锁，需等待sleeptime结束。
     * 此后a进入RUNABLE的就绪状态
     * @throws InterruptedException
     */
    public static void sleep_()throws InterruptedException{
        Object lock = new Object();

        new Thread(() -> {
            synchronized (lock) {
                for (int i = 0; i < 5; i++) {
                    System.out.println(new Date() + "\t" + Thread.currentThread().getName() + "\t" + i);
                    try { Thread.sleep(1000); } catch (InterruptedException e) { }
                }
            }
        }).start();

        Thread.sleep(1000);

        new Thread(() -> {
            synchronized (lock) {
                for (int i = 0; i < 5; i++) {
                    System.out.println(new Date() + "\t" + Thread.currentThread().getName() + "\t" + i);
                }
            }
        }).start();
    }
}

import java.util.Date;
public class NormalMethod {
    public static void main(String[] args)throws InterruptedException {
       NormalMethod.join_();

/**
     * join(timeout)，join()，分别进入Timed Waiting，Waiting，a线程中调用b线程的join方法，
     * 则a需等待b执行结束才会执行，b会让出锁，
     *join_(),在父线程中插队子线程，等待子线程执行完毕，父线程才会执行
     */
    public static void join_(){
        new Thread(new par(),"父线程").start();


    }

/**
 * 父线程
 */
class par implements Runnable{

    @Override
    public void run() {
        //创建字线程
        Thread chiTh = new Thread(new chi(),"子线程");
        //启动子线程
        chiTh.start();
//        在父线程中创建子线程，并让子线程插队
        try {
            chiTh.join();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
//        父线程任务
        for (int i = 0; i < 3; i++) {
            System.out.println(new Date()+ Thread.currentThread().getName()+"\t执行"+i);
        }


    }
}

/**
 * 子线程
 */
class chi implements Runnable{
//子线程任务
    @Override
    public void run() {
        for (int i = 0; i < 3; i++) {
            //子线程加入延时
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println(new Date()+ Thread.currentThread().getName()+"\t执行"+i);
        }

    }
